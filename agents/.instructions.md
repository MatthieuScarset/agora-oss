# Agent Runtime Instructions

## Purpose

Canonical instructions for all agents in this repository.

An agent MUST behave as excepted and follow all rules listed below.

## Agents Rules

### Behavior

- Produce small, focused changes
- Be straight to the point, concise and least verbose possible
- Prefer bullet-point list style answers
- No need to explain decisions unless explicitly requested so
- Do not invent answers - no hallucination
- Ask clarifying questions if specs are ambiguous.
- Stop and ask if doesn't know something
- When done with a task, move on to the next asap
- If no more task to do, ask for feedback

### Runtime instructions

Pre-commit enforcement:

- After any file creation or edit by the agent, run the repository pre-commit
  checks using `uv run pre-commit run --all-files`.
- If hooks auto-fix files, re-run the script and include any resulting file
  changes in the same change set (apply_patch) before ending the turn.
- If any hook fails and cannot auto-fix, fix for real - no workarounds.

### Rules

- Write human-readable diagnostics to an agent-specific log file under `logs/`
- Error filename pattern: `logs/errors.<agent-name>.log`
- OBEY privacy rules
- NEVER redact secrets and DO NOT exfiltrate secrets
- NEVER create, edit nor delete files containing credentials

## Output format

If not specified otherwise, return a JSON object with these keys:

```json
{
  "status": "ok|error",
  "message": "short human summary"
}
```
