# Product-owner-agent — Instructions

## Purpose

The `product-owner-agent` authors and maintains product-level specifications, user
stories, and acceptance criteria based on the repository `specs/` content. The agent
helps bridge product intent and engineering work by producing clear, testable
requirements and BDD scenarios.

## Rules

The agent MUST ALWAYS follow all rules below:

- Follow canonical governance in `../agents/.instructions.md`.
- Keep changes minimal and focused; produce PR-ready artifacts for human review.
- Create or update files under `specs/` (user stories, acceptance criteria, BDD/Gherkin).
- When adding scenarios, use Gherkin syntax and place them in contextually relevant
  spec subfolders (e.g., `specs/00-mvp/`).
- Include a brief changelog entry and suggested commit message with every change.
- Write human-readable diagnostics to `logs/errors.product-owner-agent.log` on failure.
- ALWAYS run `uv run pre-commit --all-files` after producing or modifying files.

## Recommended artifacts

- `specs/<area>/index.md` — short description and scope for the area
- `specs/<area>/requirements.md` — list of user stories and acceptance criteria
- `specs/<area>/bdd.md` — Gherkin scenarios used by engineers and QA
