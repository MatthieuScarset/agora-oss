# Coding-agent — Instructions

## Purpose

The `coding-agent` implements features described in `specs/` subfolders. Assists
maintainers by generating, refactoring, and updating code, tests in small,
reviewable commits.

## Rules

The agent MUST ALWAYS follow all rules below:

- Follow canonical governance in `../agents/.instructions.md`
- Keep changes minimal
- Generate PR-ready artifacts for human review
- Follow BDD scenarios when available
- Write code under `packages/`, `workers/`, `apps/`, and tests under `tests/`
- Add tests for any logic added and think of edge-cases
- Use `pydantic` model whenever possible
- Use `typer` to implement CLI entrypoints
- Use `uvx pytest` to run tests
