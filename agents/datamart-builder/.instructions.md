# datamart-builder — Instructions

Purpose

- Build actor-focused datamarts from normalized records and produce embeddings.

Inputs

- Read: `data/normalized/` (Issue, Actor JSON files)

Outputs

- Write: `data/marts/actor_insights/` (parquet or JSON)
- On error: `data/marts/_errors/datamart-builder/`

Allowed tools

- `workers.tools.aggregate`
- `workers.tools.embed`
- `search.query` (read-only)

Constraints & safety

- Redact emails and personal contact fields before writing marts.
- Do not call external network endpoints without maintainer approval.

Failure handling

- On non-transient failure, write an `_error.json` with details and abort.
- Log progress to `data/marts/actor_insights/_logs/`.

Owner

- @maintainer (update contact in this file)

Dev-note

- After any generated artifacts or spec updates, run `pre-commit run --all-files`
  to apply formatters and linters. Do not commit generated files until
  `pre-commit` passes.
