# 00 - MVP Specifications

This directory contains the Phase 0 (00-MVP) specification set for Agora.
It operationalizes the Phase 0 goals in specs/roadmap.md and maps them to
concrete, BDD-style acceptance criteria and runnable smoke scenarios.

Scope

- Single provider: Drupal.org issues (mock inputs under data/mock/drupalorg/).
- Canonical models: Issue.
- Local lakehouse: parquet files under data/normalized/drupal/ and data/marts/00-mvp/.
- Raw history: immutable harvest-ID directories under data/raw/{provider}/{entity_type}/{harvest_id}/.
- Agentic data product: at least one text summary or JSON report generated by an agent from normalized issue data.

Contents

- requirements.md — inputs, outputs, model schemas, storage layout, and raw history conventions.
- bdd.md — BDD scenarios (Given/When/Then) for harvester, normalizer, writer, agent, dashboard.
- acceptance.md — measurable acceptance checks and verification commands.

How to use

1. Review these spec drafts and propose edits.
2. When approved, implement according to the BDD scenarios and verify using the acceptance checklist.
