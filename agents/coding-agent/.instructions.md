# coding-agent — Instructions

Purpose

- The `coding-agent` is responsible for producing and modifying code artifacts
  across the Agora project. It is an implementation-focused agent intended to
  assist maintainers by generating, refactoring, and updating code, tests,
  and small scaffolding tasks.

Scope & responsibilities

- Implement features described in `specs/` and follow BDD scenarios when
  available.
- Produce small, reviewable commits and surface diffs for maintainers.
- Do not independently push commits to protected branches; generate patches
  or PR-ready artifacts for human review.

Inputs & outputs

- Read: repository files and `specs/` directory.
- Write: code under `packages/`, `workers/`, `apps/`, and tests under `tests/`.
- On error: write agent-specific human-readable diagnostics to
  `logs/errors.coding-agent.log` and a machine-readable `_error.json` to
  `data/marts/_errors/coding-agent/`. See `agents/.instructions.md` for
  the canonical logging policy.

Allowed tools

- Local workspace file I/O and lint/formatting tools defined in the repo.
- Tests via `uvx pytest` and linters via `uv run pre-commit --all-files`

Constraints & safety

- Redact secrets and do not create files containing credentials.
- Keep changes minimal, add tests for any logic added, and prefer clarity
  over cleverness.

Failure handling

- On failure, write a clear `_error.json` to `data/marts/_errors/coding-agent/`
  with the failure reason and any stack traces.

Dev-note

- This agent MUST follow the canonical agent governance in `agents/.instructions.md`:
  run `pre-commit run --all-files` after generating or modifying files, and
  implement CLI entrypoints using `typer` exposing a `main()` callable.
