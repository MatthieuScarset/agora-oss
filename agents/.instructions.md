# Agents: Global Instructions

Purpose: central, canonical instructions for all agents and automated
generators in this repository.

Key rules

- Agents MUST run `pre-commit run --all-files` after producing or
  modifying repository files. This ensures formatting, linting, and
  repository checks are applied before changes are persisted.
- Agents MUST write any error artifacts to `data/marts/_errors/` and
  surface a concise diagnostic in their output.

Inputs & outputs

- Default input path: `data/normalized/`
- Default output path: `data/marts/`

CLI guideline

- Use `typer` to implement CLI entrypoints for agents and worker
  scripts. Prefer a single `Typer()` app per module and provide a
  `main()` function that is safe to call from `python -m` for tests.

Developer notes

- When documenting agents, reference this file rather than repeating the
  requirements.
- Keep per-agent `.instructions.md` focused on agent-specific inputs,
  outputs, and constraints; do not repeat global governance rules.

Location

- This file is the canonical place for agent governance. Other agent docs and `A
GENTS.md` should link here.

Dev-checklist (for implementers)

- Run `pre-commit run --all-files` locally or via CI after code
  generation.
- Ensure CLI entrypoints use `typer` and expose a `main()` callable.
- Add tests for generated artifacts and place errors into
  `data/marts/_errors/`.

Logging

- Agents MUST write human-readable diagnostics to an agent-specific log
  file under `logs/` using the filename pattern `logs/errors.<agent-name>.log`.
- Agents MUST also produce a machine-readable `_error.json` under
  `data/marts/_errors/<agent-name>/` when an operation fails. The
  `_error.json` should contain `error`, `traceback` (if available), and
  `timestamp` fields.
